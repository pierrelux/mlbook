Modèles génératifs

Objectifs d’apprentissage

À la fin de ce chapitre, vous serez en mesure de:

  • Distinguer les approches générative et discriminative

  • Dériver l’estimateur du maximum de vraisemblance pour le classifieur naïf bayésien

  • Appliquer l’analyse discriminante gaussienne (LDA et QDA)

  • Calculer les estimateurs EMV pour ces modèles

  • Expliquer les avantages et inconvénients des modèles génératifs

Approches générative vs discriminative¶

Rappel: le théorème de Bayes¶

Pour classifier un exemple x\boldsymbol{x}x, nous voulons calculer la probabilité a posteriori:

p(y=c∣x;θ)=p(x∣y=c;θ) p(y=c;θ)∑c′p(x∣y=c′;θ) p(y=c′;θ)p(y = c \mid \boldsymbol{x}; \boldsymbol{\theta}) = \frac{p(\boldsymbol{x} \mid y = c; \boldsymbol{\theta}) \, p(y = c; \boldsymbol{\theta})}{\sum_{c'} p(\boldsymbol{x} \mid y = c'; \boldsymbol{\theta}) \, p(y = c'; \boldsymbol{\theta})}p(y=c∣x;θ)=∑c′​p(x∣y=c′;θ)p(y=c′;θ)p(x∣y=c;θ)p(y=c;θ)​
(1)

où:

  • p(x∣y=c;θ)p(\boldsymbol{x} \mid y = c; \boldsymbol{\theta})p(x∣y=c;θ) est la vraisemblance (densité conditionnelle de classe)

  • p(y=c;θ)p(y = c; \boldsymbol{\theta})p(y=c;θ) est l’a priori sur les classes

Modèles génératifs¶

Dans l’approche générative, nous modélisons explicitement:

  1. La distribution des entrées pour chaque classe: p(x∣y=c)p(\boldsymbol{x} \mid y = c)p(x∣y=c)

  2. La distribution a priori sur les classes: p(y=c)p(y = c)p(y=c)

Le nom “génératif” vient du fait que nous pouvons générer de nouvelles données:

  1. Échantillonner une classe c∼p(y)c \sim p(y)c∼p(y)

  2. Échantillonner une observation x∼p(x∣y=c)\boldsymbol{x} \sim p(\boldsymbol{x} \mid y = c)x∼p(x∣y=c)

Modèles discriminatifs¶

Dans l’approche discriminative, nous modélisons directement:

p(y=c∣x;θ)p(y = c \mid \boldsymbol{x}; \boldsymbol{\theta})p(y=c∣x;θ)
(2)

sans passer par p(x∣y)p(\boldsymbol{x} \mid y)p(x∣y). Exemples: régression logistique, SVM, réseaux de neurones.

Classifieur naïf bayésien¶

Hypothèse d’indépendance conditionnelle¶

Le classifieur naïf bayésien (Naive Bayes) suppose que les caractéristiques sont conditionnellement indépendantes étant donné la classe:

p(x∣y=c,θ)=∏d=1Dp(xd∣y=c,θdc)p(\boldsymbol{x} \mid y = c, \boldsymbol{\theta}) = \prod_{d=1}^D p(x_d \mid y = c, \boldsymbol{\theta}_{dc})p(x∣y=c,θ)=d=1∏D​p(xd​∣y=c,θdc​)
(3)

Cette hypothèse est “naïve” car elle est rarement vraie en pratique, mais le classifieur fonctionne souvent bien malgré tout.

A posteriori¶

Le a posteriori sur les classes devient:

p(y=c∣x,θ)=p(y=c∣π)∏d=1Dp(xd∣y=c,θdc)∑c′p(y=c′∣π)∏d=1Dp(xd∣y=c′,θdc′)p(y = c \mid \boldsymbol{x}, \boldsymbol{\theta}) = \frac{p(y = c \mid \boldsymbol{\pi}) \prod_{d=1}^D p(x_d \mid y = c, \boldsymbol{\theta}_{dc})}{\sum_{c'} p(y = c' \mid \boldsymbol{\pi}) \prod_{d=1}^D p(x_d \mid y = c', \boldsymbol{\theta}_{dc'})}p(y=c∣x,θ)=∑c′​p(y=c′∣π)∏d=1D​p(xd​∣y=c′,θdc′​)p(y=c∣π)∏d=1D​p(xd​∣y=c,θdc​)​
(4)

où πc=p(y=c)\pi_c = p(y = c)πc​=p(y=c) est l’a priori de classe, et les paramètres sont θ=(π,{θdc})\boldsymbol{\theta} = (\boldsymbol{\pi}, \{\boldsymbol{\theta}_{dc}\})θ=(π,{θdc​}).

EMV pour Naive Bayes¶

La vraisemblance des données s’écrit:

p(D∣θ)=∏n=1NCat(yn∣π)∏d=1Dp(xnd∣yn,θd)p(\mathcal{D} \mid \boldsymbol{\theta}) = \prod_{n=1}^N \text{Cat}(y_n \mid \boldsymbol{\pi}) \prod_{d=1}^D p(x_{nd} \mid y_n, \boldsymbol{\theta}_d)p(D∣θ)=n=1∏N​Cat(yn​∣π)d=1∏D​p(xnd​∣yn​,θd​)
(5)

La log-vraisemblance se factorise:

log⁡p(D∣θ)=log⁡p(Dy∣π)+∑c∑dlog⁡p(Ddc∣θdc)\log p(\mathcal{D} \mid \boldsymbol{\theta}) = \log p(\mathcal{D}_y \mid \boldsymbol{\pi}) + \sum_c \sum_d \log p(\mathcal{D}_{dc} \mid \boldsymbol{\theta}_{dc})logp(D∣θ)=logp(Dy​∣π)+c∑​d∑​logp(Ddc​∣θdc​)
(6)

où Dy={yn}\mathcal{D}_y = \{y_n\}Dy​={yn​} sont les étiquettes et Ddc={xnd:yn=c}\mathcal{D}_{dc} = \{x_{nd} : y_n = c\}Ddc​={xnd​:yn​=c} sont les valeurs du trait ddd pour les exemples de classe ccc.

Cela permet d’optimiser chaque terme séparément.

Cas des caractéristiques catégorielles¶

Si les traits prennent des valeurs discrètes parmi KKK possibilités, l’EMV est:

θ^dck=NdckNc\hat{\theta}_{dck} = \frac{N_{dck}}{N_c}θ^dck​=Nc​Ndck​​
(7)

où Ndck=∑n=1NI(xnd=k,yn=c)N_{dck} = \sum_{n=1}^N \mathbb{I}(x_{nd} = k, y_n = c)Ndck​=∑n=1N​I(xnd​=k,yn​=c) est le nombre de fois que le trait ddd vaut kkk parmi les exemples de classe ccc.

Cas des caractéristiques binaires¶

Si les traits sont binaires (présent/absent), on utilise la loi de Bernoulli:

θ^dc=NdcNc\hat{\theta}_{dc} = \frac{N_{dc}}{N_c}θ^dc​=Nc​Ndc​​
(8)

où NdcN_{dc}Ndc​ est le nombre de fois que le trait ddd est présent dans la classe ccc.

Lissage de Laplace¶

Un problème survient si un trait n’est jamais observé avec une certaine classe: Ndc=0N_{dc} = 0Ndc​=0 implique θ^dc=0\hat{\theta}_{dc} = 0θ^dc​=0, ce qui rend p(x∣y=c)=0p(\boldsymbol{x} \mid y = c) = 0p(x∣y=c)=0 même si un seul trait est absent.

Le lissage de Laplace (add-one smoothing) résout ce problème:

θ^dc=Ndc+1Nc+K\hat{\theta}_{dc} = \frac{N_{dc} + 1}{N_c + K}θ^dc​=Nc​+KNdc​+1​
(9)

où KKK est le nombre de valeurs possibles. Cela correspond à un a priori uniforme dans le cadre MAP.

Analyse discriminante gaussienne¶

Modèle¶

L’analyse discriminante gaussienne (GDA, Gaussian Discriminant Analysis) suppose que les densités conditionnelles de classe sont gaussiennes:

p(x∣y=c,θ)=N(x∣μc,Σc)p(\boldsymbol{x} \mid y = c, \boldsymbol{\theta}) = \mathcal{N}(\boldsymbol{x} \mid \boldsymbol{\mu}_c, \boldsymbol{\Sigma}_c)p(x∣y=c,θ)=N(x∣μc​,Σc​)
(10)

Le a posteriori sur les classes est:

p(y=c∣x,θ)∝πcN(x∣μc,Σc)p(y = c \mid \boldsymbol{x}, \boldsymbol{\theta}) \propto \pi_c \mathcal{N}(\boldsymbol{x} \mid \boldsymbol{\mu}_c, \boldsymbol{\Sigma}_c)p(y=c∣x,θ)∝πc​N(x∣μc​,Σc​)
(11)

Log a posteriori et fonction discriminante¶

Le logarithme du a posteriori (appelé fonction discriminante) est:

log⁡p(y=c∣x,θ)=log⁡πc−12log⁡∣2πΣc∣−12(x−μc)⊤Σc−1(x−μc)+const\log p(y = c \mid \boldsymbol{x}, \boldsymbol{\theta}) = \log \pi_c - \frac{1}{2}\log|2\pi\boldsymbol{\Sigma}_c| - \frac{1}{2}(\boldsymbol{x} - \boldsymbol{\mu}_c)^\top \boldsymbol{\Sigma}_c^{-1}(\boldsymbol{x} - \boldsymbol{\mu}_c) + \text{const}logp(y=c∣x,θ)=logπc​−21​log∣2πΣc​∣−21​(x−μc​)⊤Σc−1​(x−μc​)+const
(12)

Le terme quadratique (x−μc)⊤Σc−1(x−μc)(\boldsymbol{x} - \boldsymbol{\mu}_c)^\top \boldsymbol{\Sigma}_c^{-1}(\boldsymbol{x} - \boldsymbol{\mu}_c)(x−μc​)⊤Σc−1​(x−μc​) est la distance de Mahalanobis entre x\boldsymbol{x}x et μc\boldsymbol{\mu}_cμc​.

Analyse discriminante quadratique (QDA)¶

Quand chaque classe a sa propre matrice de covariance Σc\boldsymbol{\Sigma}_cΣc​, la frontière de décision est quadratique. Cette méthode s’appelle QDA (Quadratic Discriminant Analysis).

Les lignes de niveau sont des ellipsoïdes dont l’orientation et les axes sont déterminés par les vecteurs propres de Σc\boldsymbol{\Sigma}_cΣc​.

Analyse discriminante linéaire (LDA)¶

Si toutes les classes partagent la même covariance Σ\boldsymbol{\Sigma}Σ, le terme quadratique se simplifie:

log⁡p(y=c∣x,θ)=log⁡πc−12μc⊤Σ−1μc⏟γc+x⊤Σ−1μc⏟βc+κ\log p(y = c \mid \boldsymbol{x}, \boldsymbol{\theta}) = \underbrace{\log \pi_c - \frac{1}{2}\boldsymbol{\mu}_c^\top \boldsymbol{\Sigma}^{-1} \boldsymbol{\mu}_c}_{\gamma_c} + \boldsymbol{x}^\top \underbrace{\boldsymbol{\Sigma}^{-1} \boldsymbol{\mu}_c}_{\boldsymbol{\beta}_c} + \kappalogp(y=c∣x,θ)=γc​logπc​−21​μc⊤​Σ−1μc​​​+x⊤βc​Σ−1μc​​​+κ
(13)

où κ\kappaκ ne dépend pas de ccc. La fonction discriminante est linéaire en x\boldsymbol{x}x:

log⁡p(y=c∣x,θ)=γc+x⊤βc+κ\log p(y = c \mid \boldsymbol{x}, \boldsymbol{\theta}) = \gamma_c + \boldsymbol{x}^\top \boldsymbol{\beta}_c + \kappalogp(y=c∣x,θ)=γc​+x⊤βc​+κ
(14)

Cette méthode s’appelle LDA (Linear Discriminant Analysis).

Estimation des paramètres¶

Pour la GDA, l’EMV des paramètres est:

A priori de classe:

π^c=NcN\hat{\pi}_c = \frac{N_c}{N}π^c​=NNc​​
(15)

Moyenne par classe:

μ^c=1Nc∑n:yn=cxn\hat{\boldsymbol{\mu}}_c = \frac{1}{N_c} \sum_{n: y_n = c} \boldsymbol{x}_nμ^​c​=Nc​1​n:yn​=c∑​xn​
(16)

Covariance par classe (QDA):

Σ^c=1Nc∑n:yn=c(xn−μ^c)(xn−μ^c)⊤\hat{\boldsymbol{\Sigma}}_c = \frac{1}{N_c} \sum_{n: y_n = c} (\boldsymbol{x}_n - \hat{\boldsymbol{\mu}}_c)(\boldsymbol{x}_n - \hat{\boldsymbol{\mu}}_c)^\topΣ^c​=Nc​1​n:yn​=c∑​(xn​−μ^​c​)(xn​−μ^​c​)⊤
(17)

Covariance partagée (LDA):

Σ^=1N∑c=1C∑n:yn=c(xn−μ^c)(xn−μ^c)⊤\hat{\boldsymbol{\Sigma}} = \frac{1}{N} \sum_{c=1}^C \sum_{n: y_n = c} (\boldsymbol{x}_n - \hat{\boldsymbol{\mu}}_c)(\boldsymbol{x}_n - \hat{\boldsymbol{\mu}}_c)^\topΣ^=N1​c=1∑C​n:yn​=c∑​(xn​−μ^​c​)(xn​−μ^​c​)⊤
(18)

Avantages des modèles génératifs¶

  1. Apprentissage parfois plus simple: Naive Bayes s’apprend par comptage et moyennes, sans optimisation itérative

  2. Gestion des caractéristiques manquantes: On peut marginaliser sur les valeurs manquantes

  3. Apprentissage incrémental par classe: On peut ajouter de nouvelles classes sans réentraîner les existantes

  4. Apprentissage semi-supervisé: On peut utiliser des données non étiquetées pour améliorer l’estimation de p(x)p(\boldsymbol{x})p(x)

  5. Robustesse aux caractéristiques factices: Modéliser p(x∣y)p(\boldsymbol{x} \mid y)p(x∣y) peut capturer les mécanismes causaux sous-jacents

Inconvénients des modèles génératifs¶

  1. Hypothèses plus fortes: Le modèle génératif doit spécifier p(x∣y)p(\boldsymbol{x} \mid y)p(x∣y), ce qui est plus contraignant

  2. Précision souvent inférieure: Les modèles discriminatifs optimisent directement ce qui nous intéresse

  3. Difficultés avec les caractéristiques prétraitées: Après extraction de caractéristiques, les nouvelles variables peuvent avoir des corrélations complexes difficiles à modéliser

  4. Probabilités mal calibrées: Les hypothèses fortes (comme l’indépendance) peuvent donner des probabilités biaisées

Résumé¶

Les modèles génératifs modélisent la distribution jointe des entrées et des étiquettes:

  • Le classifieur naïf bayésien suppose l’indépendance conditionnelle des caractéristiques

  • L’analyse discriminante gaussienne modélise les classes par des gaussiennes

  • LDA utilise une covariance partagée (frontière linéaire)

  • QDA permet des covariances différentes (frontière quadratique)

Ces modèles sont simples à entraîner (formules fermées pour l’EMV), interprétables, et permettent de générer des données synthétiques.

Exercices¶

Exercice 1: Naive Bayes binaire

Un classifieur Naive Bayes est entraîné sur des emails (spam vs non-spam) avec 2 traits binaires: présence du mot “gratuit” (x1x_1x1​) et présence du mot “urgent” (x2x_2x2​).

Données d’entraînement:

  • Spam (10 emails): 8 avec “gratuit”, 6 avec “urgent”

  • Non-spam (20 emails): 2 avec “gratuit”, 4 avec “urgent”

  1. Calculez les EMV de tous les paramètres.

  2. Classifiez un email contenant “gratuit” mais pas “urgent”.

  3. Appliquez le lissage de Laplace et recalculez.

Exercice 2: LDA en 1D

Deux classes en 1D:

  • Classe 0: μ0=0\mu_0 = 0μ0​=0, exemples: {−1,0,1}\{-1, 0, 1\}{−1,0,1}

  • Classe 1: μ1=3\mu_1 = 3μ1​=3, exemples: {2,3,4}\{2, 3, 4\}{2,3,4}

Avec une covariance partagée σ2=1\sigma^2 = 1σ2=1 et des a priori égaux:

  1. Écrivez la fonction discriminante pour chaque classe.

  2. Trouvez le seuil de décision.

  3. Classifiez x=1.5x = 1.5x=1.5.

Exercice 3: QDA vs LDA

Expliquez géométriquement pourquoi:

  1. LDA produit des frontières de décision linéaires

  2. QDA produit des frontières de décision quadratiques

  3. Donnez un exemple de données où QDA serait nettement meilleur que LDA

Exercice 4: Distance de Mahalanobis

Soit μ=(0,0)\boldsymbol{\mu} = (0, 0)μ=(0,0) et Σ=(4001)\boldsymbol{\Sigma} = \begin{pmatrix} 4 & 0 \\ 0 & 1 \end{pmatrix}Σ=(40​01​).

  1. Calculez la distance de Mahalanobis de (2,0)(2, 0)(2,0) à μ\boldsymbol{\mu}μ.

  2. Calculez la distance de Mahalanobis de (0,2)(0, 2)(0,2) à μ\boldsymbol{\mu}μ.

  3. Pourquoi ces distances sont-elles différentes malgré la même distance euclidienne?
