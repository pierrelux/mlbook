Annexe: Révision d’optimisation

Cette annexe révise les concepts d’optimisation utilisés tout au long du livre.

Formulation générale¶

Un problème d’optimisation s’écrit:

min⁡x∈Xf(x)\min_{\boldsymbol{x} \in \mathcal{X}} f(\boldsymbol{x})x∈Xmin​f(x)
(1)

où:

  • f:RD→Rf: \mathbb{R}^D \to \mathbb{R}f:RD→R est la fonction objectif

  • X⊆RD\mathcal{X} \subseteq \mathbb{R}^DX⊆RD est l’ensemble réalisable

  • x∗\boldsymbol{x}^*x∗ est un minimiseur (ou optimum)

Note: Maximiser fff équivaut à minimiser −f-f−f.

Convexité¶

Ensembles convexes¶

Un ensemble C\mathcal{C}C est convexe si pour tout x,y∈C\boldsymbol{x}, \boldsymbol{y} \in \mathcal{C}x,y∈C et α∈[0,1]\alpha \in [0, 1]α∈[0,1]:

αx+(1−α)y∈C\alpha \boldsymbol{x} + (1-\alpha)\boldsymbol{y} \in \mathcal{C}αx+(1−α)y∈C
(2)

Intuition: le segment entre deux points de l’ensemble reste dans l’ensemble.

Fonctions convexes¶

Une fonction fff est convexe si son domaine est convexe et:

f(αx+(1−α)y)≤αf(x)+(1−α)f(y)f(\alpha \boldsymbol{x} + (1-\alpha)\boldsymbol{y}) \leq \alpha f(\boldsymbol{x}) + (1-\alpha) f(\boldsymbol{y})f(αx+(1−α)y)≤αf(x)+(1−α)f(y)
(3)

pour tout α∈[0,1]\alpha \in [0, 1]α∈[0,1].

Interprétation géométrique: la fonction est en-dessous de toute corde.

Caractérisations¶

Pour une fonction fff deux fois différentiable:

ConditionSignification
fff convexe∇2f(x)⪰0\nabla^2 f(\boldsymbol{x}) \succeq 0∇2f(x)⪰0 (Hessienne semi-définie positive)
fff strictement convexe∇2f(x)≻0\nabla^2 f(\boldsymbol{x}) \succ 0∇2f(x)≻0 (Hessienne définie positive)
fff concave−f-f−f convexe

Exemples de fonctions convexes¶

  • Fonctions affines: f(x)=a⊤x+bf(\boldsymbol{x}) = \boldsymbol{a}^\top \boldsymbol{x} + bf(x)=a⊤x+b

  • Normes: f(x)=∥x∥pf(\boldsymbol{x}) = \|\boldsymbol{x}\|_pf(x)=∥x∥p​ pour p≥1p \geq 1p≥1

  • Formes quadratiques: f(x)=x⊤Axf(\boldsymbol{x}) = \boldsymbol{x}^\top \mathbf{A} \boldsymbol{x}f(x)=x⊤Ax si A⪰0\mathbf{A} \succeq 0A⪰0

  • Exponentielle: f(x)=exf(x) = e^xf(x)=ex

  • Entropie négative: f(x)=xlog⁡xf(x) = x \log xf(x)=xlogx pour x>0x > 0x>0

Propriété fondamentale¶

Pour une fonction convexe, tout minimum local est un minimum global.

C’est pourquoi les problèmes convexes sont plus faciles à résoudre.

Conditions d’optimalité¶

Condition nécessaire du premier ordre¶

Si x∗\boldsymbol{x}^*x∗ est un minimum local et fff est différentiable:

∇f(x∗)=0\nabla f(\boldsymbol{x}^*) = \mathbf{0}∇f(x∗)=0
(4)

Un point où ∇f=0\nabla f = \mathbf{0}∇f=0 est appelé point stationnaire.

Condition nécessaire du second ordre¶

Si x∗\boldsymbol{x}^*x∗ est un minimum local et fff est deux fois différentiable:

∇f(x∗)=0et∇2f(x∗)⪰0\nabla f(\boldsymbol{x}^*) = \mathbf{0} \quad \text{et} \quad \nabla^2 f(\boldsymbol{x}^*) \succeq 0∇f(x∗)=0et∇2f(x∗)⪰0
(5)

Condition suffisante du second ordre¶

Si ∇f(x∗)=0\nabla f(\boldsymbol{x}^*) = \mathbf{0}∇f(x∗)=0 et ∇2f(x∗)≻0\nabla^2 f(\boldsymbol{x}^*) \succ 0∇2f(x∗)≻0, alors x∗\boldsymbol{x}^*x∗ est un minimum local strict.

Points-selle¶

Un point-selle satisfait ∇f=0\nabla f = \mathbf{0}∇f=0 mais n’est ni un minimum ni un maximum:

  • Minimum dans certaines directions

  • Maximum dans d’autres directions

La Hessienne a des valeurs propres de signes différents.

Descente de gradient¶

Algorithme de base¶

Mise à jour itérative:

x(t+1)=x(t)−η∇f(x(t))\boldsymbol{x}^{(t+1)} = \boldsymbol{x}^{(t)} - \eta \nabla f(\boldsymbol{x}^{(t)})x(t+1)=x(t)−η∇f(x(t))
(6)

où η>0\eta > 0η>0 est le taux d’apprentissage (learning rate).

Intuition: on se déplace dans la direction de plus forte descente.

Choix du taux d’apprentissage¶

  • η\etaη trop petit: convergence lente

  • η\etaη trop grand: oscillations, divergence possible

Règle pratique: commencer avec η\etaη modéré, observer la perte.

Convergence¶

Pour une fonction LLL-lisse (∥∇f(x)−∇f(y)∥≤L∥x−y∥\|\nabla f(\boldsymbol{x}) - \nabla f(\boldsymbol{y})\| \leq L\|\boldsymbol{x} - \boldsymbol{y}\|∥∇f(x)−∇f(y)∥≤L∥x−y∥) et convexe:

  • Avec η≤1/L\eta \leq 1/Lη≤1/L: convergence en O(1/T)O(1/T)O(1/T) itérations

  • Si fortement convexe: convergence linéaire en O(κlog⁡(1/ϵ))O(\kappa \log(1/\epsilon))O(κlog(1/ϵ))

où κ\kappaκ est le conditionnement (rapport des valeurs propres extrêmes de la Hessienne).

Pour une fonction de la forme:

À chaque itération, on utilise un estimateur non biaisé du gradient:

Méthodes avec momentum¶

min⁡xf(x)s.t.gi(x)≤0,hj(x)=0\min_{\boldsymbol{x}} f(\boldsymbol{x}) \quad \text{s.t.} \quad g_i(\boldsymbol{x}) \leq 0, \quad h_j(\boldsymbol{x}) = 0xmin​f(x)s.t.gi​(x)≤0,hj​(x)=0

Conditions KKT (Karush-Kuhn-Tucker)¶

Pour le problème général avec inégalités:

Le Lagrangien:

L(x,μ,λ)=f(x)+∑iμigi(x)+∑jλjhj(x)\mathcal{L}(\boldsymbol{x}, \boldsymbol{\mu}, \boldsymbol{\lambda}) = f(\boldsymbol{x}) + \sum_i \mu_i g_i(\boldsymbol{x}) + \sum_j \lambda_j h_j(\boldsymbol{x})L(x,μ,λ)=f(x)+i∑​μi​gi​(x)+j∑​λj​hj​(x)

Conditions KKT (nécessaires pour l’optimalité):

  1. Stationnarité: ∇xL=0\nabla_{\boldsymbol{x}} \mathcal{L} = \mathbf{0}∇x​L=0

  2. Réalisabilité primale: gi(x∗)≤0g_i(\boldsymbol{x}^*) \leq 0gi​(x∗)≤0, hj(x∗)=0h_j(\boldsymbol{x}^*) = 0hj​(x∗)=0

  3. Réalisabilité duale: μi≥0\mu_i \geq 0μi​≥0

  4. Complémentarité: μigi(x∗)=0\mu_i g_i(\boldsymbol{x}^*) = 0μi​gi​(x∗)=0

  • Si gi(x∗)<0g_i(\boldsymbol{x}^*) < 0gi​(x∗)<0 (contrainte inactive): μi=0\mu_i = 0μi​=0

  • Si μi>0\mu_i > 0μi​>0: gi(x∗)=0g_i(\boldsymbol{x}^*) = 0gi​(x∗)=0 (contrainte active)

Le problème primal:

Méthodes de Newton¶

Newton classique¶

Utilise l’information de second ordre:

x(t+1)=x(t)−[∇2f(x(t))]−1∇f(x(t))\boldsymbol{x}^{(t+1)} = \boldsymbol{x}^{(t)} - [\nabla^2 f(\boldsymbol{x}^{(t)})]^{-1} \nabla f(\boldsymbol{x}^{(t)})x(t+1)=x(t)−[∇2f(x(t))]−1∇f(x(t))
(32)

Avantages:

  • Convergence quadratique près de l’optimum

  • Invariant aux transformations affines

Inconvénients:

  • Coût O(D3)O(D^3)O(D3) pour l’inversion

  • Peut diverger si la Hessienne n’est pas définie positive

Quasi-Newton (BFGS, L-BFGS)¶

Approxime la Hessienne sans la calculer explicitement.

L-BFGS (Limited-memory BFGS):

  • Stocke seulement les mmm dernières différences de gradient

  • Coût O(mD)O(mD)O(mD) par itération

  • Très utilisé pour l’optimisation de grande dimension
